Why Linux?

After using Windows most of my life, I chose to move to Linux because I am sick and tired of the new direction of Windows. Unfortunately Linux isn't perfect either, it is much more fragile and does not work as well with proprietary stuff.

I am now in the stage of my life where I just want things to work. I don't want to spend time debugging, configuring and fixing issues. Life is too short, I want my computer to work whenever I need it so that I can focus on projects that I find interesting.

This is why my setup is now:

  • Desktop with Fedora for programming projects
  • Macbook with MacOS for casual stuff (web, email, video streaming, pictures...)

Philosophy

After years of working in the software industry I have realized that software can break in the most absurd stupid ways due to thing that should have no link to each others but yet they somehow do. Absolutely anything and everything can break it and Murphy's Law says that if there is a way for something to break it will. The universe is against you on this, the second law of thermodynamics says that entropy only increases.

When we talk about "Linux" as an OS, we mean a linux distribution, meaning the Linux kernel with a bunch of packages and apps "taped" together.

I said that Linux is fragile, this is probably the main reason. It is not an OS but a collection of OS sharing most stuff but each one is annoyingly a little bit different from each other. Everything is highly customizable, no one has the same configuration, leading to untested use cases and bugs!

On top of that, big components of the OS are developped by wastly different teams of people from all over the world with sometimes hard philosophical opinions, leading to harsh disagreements and even project forks. No wonder it sometimes feels like an inconsistent mess.

This is why I now try to use everything by default. This is the most tested configuration which in theory should have lower chance of stupid bugs.

Why Fedora?

Fedora

  • set it and forget, just works
  • popular
    • used by kernel devs (Linus Torvalds)
  • backed by Red Hat
    • commercial support
    • upstream of RHEL
    • major contributor of kernel, gnome, wayland, systemd, pulseaudio, kvm...
  • target audience seems to be devs
  • has the newest stuff first
  • packages manager is slow

Ubuntu

  • most popular, most commercial support
  • target everyone
    • going the same way as Windows (corporate BS)
  • strange bugs
  • outdated packages
  • not a fan of Unity interface

Arch

  • great documentation
  • pacman
  • AUR
  • rolling-release
    • in theory this is great but in practice it's annoying
  • too much setup
  • too much maintenance
  • too much customization -> bugs

After using it daily for a few years, it is not for me.

Why Gnome?

  • Gnome
    • pretty & finished
    • default for Fedora (and many other)
    • best wayland support
    • new UI is strange, need extensions like Dash to Panel and Arc Menu
    • strong opinions (no tray icons seriously??)
  • KDE
    • full featured
    • ugly, messy, bloated UI
    • wayland support is less good
  • Xfce
    • light & fast
    • missing basic features
    • doesn't feel finished
    • no wayland support
    • not as many users, not as many devs -> bugs

Why Windows sucks

  • it just works
  • best hardware compatibility by far
  • most commercial support
  • great for games

As a dev environment it is not that great:

  • reboot automatically after updates and no way to disable it
  • updates that need multiple reboots to install more updates
  • install & update programs (setup.exe)
    • winget now but not everything is in there and doesn't always work
  • install dev libs
    • vcpkg now
  • terminal & shell is not that nice
    • Windows Terminal now but it's slow as hell
    • WSL but its not the same, it's a weird mix with weirds bugs, runs in a VM

Even for casual stuff it is becoming a massive pain to use:

  • telemetry
  • bloated
    • cortana
    • AI
    • web apps
  • stupid UI changes -> messy inconsistent UI
    • mix of XP old ones, new ones, new new ones..
  • slow apps
    • the new UI apps are slow as hell, the new image viewer is a disaster
  • arbitrary hardware requirements (programmed obsolescence)

Why MacOS sucks

  • pretty & cohesive
  • easy
  • great integration with other Apple products
  • better privacy than most other tech companies
  • great hardware, battery life
  • pain to develop on
    • feels like Unix but not Unix, tons of little incompatible annoyances
    • strong opinions, "the Apple way"
    • not a big fan of homebrew, not official, can break things
    • Xcode LOL!
  • buggy, crashes (yes even with latest HW & SW doing casual stuff...)
  • closed ecosystem (golden jail)
    • they do a very good job at making sure you only use their products and not the one of competitors..
  • programmed obsolescence
  • expensive hardware

Why Linux sucks

  • free
  • open source
  • light & fast
  • nice dev environment
  • buggy, fragile, things often break
  • inconsistent: many distributions, many desktop environments
  • distributing binaries is very hard
    • now there is flatpak, snap and AppImage (great now we have 3 standards...)
  • proprietary apps might not work well
  • UI apps are often ugly
